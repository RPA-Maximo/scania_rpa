文档二：RPA 自动入库终极引擎说明
工具名称
rpa_sync_maximo_to_db.py

🛠️ 工具作用
这是端到端的终极 RPA 自动化管线。该脚本不仅集成了 export_wos_to_json.py 的所有极速抓取和清洗能力，还额外增加了直连 MySQL 数据库的功能。它会自动在目标数据库中创建表结构（纯英文字段），采用“主表 UPSERT 更新，子表先删后插”的幂等写入策略，确保你可以无限次重复运行该脚本而不会导致数据库出现重复脏数据。同时，它会在本地留存一份 JSON 备份作为审计日志。
📦 前置准备
确保安装了相关依赖：uv pip install requests pymysql

确保拥有 wo_config.txt 配置文件。

数据库网络畅通：确保运行该脚本的电脑能够访问配置的 MySQL 数据库 IP（222.187.11.98）。
⚙️ 命令行参数说明
(参数配置与 export_wos_to_json.py 完全一致，无缝切换)
参数,简写,是否必填,说明
--status,-s,必填,工单状态 (如：COMP)
--pages,-p,选填,抓取页数 (默认 3)
--size,-n,选填,每页条数 (默认 10)
--start,无,选填,开始日期 (默认 2025-07-01)
--end,无,选填,结束日期 (默认 2025-07-31)
💻 运行示例
标准数据同步作业（拉取并自动落库）

PowerShell
uv run .\rpa_sync_maximo_to_db.py -s COMP -p 3 -n 10 --start 2025-07-01 --end 2025-07-31
🗄️ 数据库表结构说明
脚本会自动在数据库中维护以下两张表：

work_order (主表)：存放工单号、描述、总科目、成本中心、状态、审批人等主线信息。

work_order_cost (子表)：存放关联该工单的所有明细费用项（通过 wonum 外键关联），包含费用类型、工种标识和具体金额。
